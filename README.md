# HackJos 2025 - Igniting MSME Growth Through Innovation

HackJos 2025 is a premier hackathon event organized by nHub Nigeria to celebrate 10 years of innovation and entrepreneurship. This repository contains essential documentation and resources for participating teams to build viable MVPs (Minimum Viable Products) that demonstrate **innovation**, **usability**, and **scalability**.

## Challenge Tracks

Participants must choose one of the following tracks:

### 1. E-commerce Solutions

Build solutions that enable small businesses or individuals to buy, sell, and trade products or services more efficiently in Nigeria and across Africa.

**Problem Areas:**

- Digital storefront creation for local merchants
- Secure and simplified online payment integration
- Product discovery and personalization
- Community marketplaces (social commerce)
- Integrating offline sellers into online systems

### 2. Financial Inclusion

Create tools that make financial services more accessible to underserved individuals and communities.

**Problem Areas:**

- Simplified mobile banking or savings systems
- Micro-lending, group saving (ajo/esusu) tools
- Financial literacy or credit scoring solutions
- Offline transaction handling for low-connectivity users

### 3. Productivity Tools

Build apps that enhance personal or organizational productivity ‚Äî helping people work smarter, collaborate better, or automate routine tasks.

**Problem Areas:**

- Task management or workflow automation
- Team communication and collaboration tools
- Time tracking, goal setting, or habit tracking
- AI-assisted productivity or summarization tools

### 4. Logistics & Last-mile Delivery

Develop systems that improve transportation, delivery tracking, and supply-chain efficiency, especially for local and regional operations.

**Problem Areas:**

- Route optimization for drivers
- Package tracking and proof of delivery
- Real-time map visualization and ETA updates
- Dispatch systems for delivery companies

## General Technical Requirements

All submissions **must**:

1. ‚úÖ **Be functional** - The MVP should run and demonstrate its core workflow (even if limited)
2. ‚úÖ **Have a clear user journey** - Users should be able to navigate through the main use case with minimal explanation
3. ‚úÖ **Use modern technology** - Web (React, Next.js, Vue, Angular), Mobile (React Native, Flutter, Kotlin, Swift), or API-based systems (Node.js, Django, etc.)
4. ‚úÖ **Show version control** - Projects must be hosted on GitHub (public or private repo with access granted to judges)
5. ‚úÖ **Include documentation** - A README.md file explaining:
   - What the product does
   - Setup instructions
   - Demo credentials (if applicable)
6. ‚úÖ **Demonstrate deployment readiness** - A hosted version on platforms like Vercel, Netlify, Render, or Firebase
7. ‚úÖ **Be team-built** - Use collaboration tools like GitHub, Trello, Figma; judges may ask to see proof of contribution balance

## Technology Stack Recommendations

- **Web:** React, Next.js, Vue.js, Angular

### Backend

- **Languages/Frameworks:** Node.js, Python (Django/Flask), PHP (Laravel), Ruby on Rails
